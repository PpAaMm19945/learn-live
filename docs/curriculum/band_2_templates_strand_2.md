### B2: Unknown Quantities
**Prerequisite:** B1 / D8 (Equivalence)

#### Cognitive Level 1: Encounter
| Field | Value |
|---|---|
| **Task type** | Number riddle (Oral/Physical) |
| **Materials** | A small cup to hide objects |
| **Parent Prompt** | "Hide {N} objects under a cup. Show {M} objects outside. Tell your child: 'Altogether, there are {Total} objects. How many are hiding?'" |
| **Success** | Child finds the hidden amount without lifting the cup first. Says "because {N} and {M} make {Total}". |

#### Cognitive Level 2: Execute
| Field | Value |
|---|---|
| **Task type** | Symbolic representation |
| **Repetition Arc** | **Execution (x5):** Give missing number equations: "? + 15 = 25". Use shapes or letters instead of '?'.<br>**Endurance (x2):** "I have two bags with the SAME number of apples (2x), plus 3 outside. I have 11 total. How many in each bag?" (Intro to 2x + 3 = 11 informally). |
| **Parent Prompt** | "Show your child equations with a missing piece represented by a box (Box - {Sub} = {Diff}). Ask them to find the box without guessing randomly." |
| **Success** | Uses inverse operations (e.g., addition to solve subtraction) to find the unknown. |

#### Cognitive Level 3: Discern
| Field | Value |
|---|---|
| **Task type** | Constraints & Possibilities |
| **Parent Prompt** | "Show: Triangle + Square = 10. Ask: 'If Triangle is 4, what is Square? Can Triangle be 12? Why or why not?'" |
| **Success** | Understands that variables can take different values, but are constrained by the equation (if one goes up, the other must go down to keep the balance of 10). |

#### Cognitive Level 4: Own
| Field | Value |
|---|---|
| **Task type** | Riddle Generation |
| **Parent Prompt** | "Say: 'Write a number riddle for me where I have to figure out a secret number you are thinking of.' You must use two operations (like add and subtract)." |
| **Success** | Creates a solvable, multi-step riddle (e.g., "I multiply by 2, then add 1, and get 11. What's my number?"). |

### B3: Function Machines
**Prerequisite:** B2 (Unknown Quantities), D2 (Addition)

#### Cognitive Level 1: Encounter
| Field | Value |
|---|---|
| **Task type** | Physical or visual "machine" |
| **Materials** | A cardboard box with an "In" and "Out" slot, or drawing |
| **Parent Prompt** | "Say: 'This is a magic machine. If I put in 2, it spits out 5. If I put in 4, it spits out 7. What happens if I put in 10?'" |
| **Success** | Discovers the rule is "+3" and applies it correctly to 10. |

#### Cognitive Level 2: Execute
| Field | Value |
|---|---|
| **Task type** | Input/Output Tables |
| **Repetition Arc** | **Execution (x5):** Provide IN/OUT tables where the rule is given. Child fills in the OUT values. Then tables where OUT is given, child finds IN.<br>**Endurance (x3):** Provide a table where the inputs are out of sequence (2, 5, 8, 1) and child must find the rule AND the missing outputs. |
| **Parent Prompt** | "Give your child an In/Out table. Ask them to find the missing numbers. Make sure they test their rule on EVERY row to make sure it works." |
| **Success** | Correctly identifies constant rate rules (+, -, or x) and applies them forward and backward. |

#### Cognitive Level 3: Discern
| Field | Value |
|---|---|
| **Task type** | Counterexample & Falsification |
| **Parent Prompt** | "Show a table: IN(2)->OUT(4). Your friend says 'The rule is +2.' Do you agree? Show me another row that proves the rule could be 'multiply by 2' instead." |
| **Success** | Recognizes that a single data point is not enough to define a linear function. Provides an IN(3)->OUT(6) row to prove multiplication, or IN(3)->OUT(5) for addition. |

#### Cognitive Level 4: Own
| Field | Value |
|---|---|
| **Task type** | Rule Design & Obfuscation |
| **Parent Prompt** | "Say: 'Design a secret function machine rule. Give me 3 inputs and outputs, and see if I can guess it.' Try to make it tricky using subtraction." |
| **Success** | Creates a mathematically sound IN/OUT table and successfully validates the parent's answer. |

### B4: Generalization from Arithmetic
**Prerequisite:** D7 (Properties of Operations)

#### Cognitive Level 1: Encounter
| Field | Value |
|---|---|
| **Task type** | Pattern recognition in numbers |
| **Parent Prompt** | "Ask your child: 'What happens when you add zero to 5? To 100? To a million?'" |
| **Success** | States that adding zero always gives you the number you started with (Identity Property of Addition). |

#### Cognitive Level 2: Execute
| Field | Value |
|---|---|
| **Task type** | Testing claims |
| **Repetition Arc** | **Execution (x4):** Test specific claims: "When you multiply any number by 1, what happens?" "When you subtract a number from itself, what happens?"<br>**Endurance (x2):** "Someone says adding two odd numbers ALWAYS makes an even number. Test it with 5 different pairs." |
| **Parent Prompt** | "Give your child the claim: '{Claim}'. Ask them to test it with small numbers, big numbers, and zero." |
| **Success** | Uses multiple test cases to verify a mathematical generalization. |

#### Cognitive Level 3: Discern
| Field | Value |
|---|---|
| **Task type** | Falsification / Counterexamples |
| **Parent Prompt** | "Ask: 'Your friend says: When you multiply two numbers, the answer is ALWAYS bigger than both numbers. Is that true? Try to find just ONE example that proves them wrong.'" |
| **Success** | Finds a counterexample: multiplying by 1 (4 x 1 = 4, not bigger) or by 0 (4 x 0 = 0). |

#### Cognitive Level 4: Own
| Field | Value |
|---|---|
| **Task type** | Rule formulation |
| **Parent Prompt** | "Ask: 'Look at a calendar. Pick any 2x2 square of numbers. Add the diagonals. Try it on another square. Write a rule for what you found.'" |
| **Success** | Formulates a general rule (the sums of the diagonals in a 2x2 calendar square are always equal) based on specific cases. |

### B5: Growing Patterns
**Prerequisite:** A1 (Pattern Recognition)

#### Cognitive Level 1: Encounter
| Field | Value |
|---|---|
| **Task type** | Visual progression |
| **Materials** | Blocks, coins, or drawn shapes |
| **Parent Prompt** | "Build step 1 (1 block), step 2 (3 blocks in a triangle), step 3 (6 blocks). Ask: 'How is this pattern growing? Build step 4.'" |
| **Success** | Sees that each step adds a row with one more block than the last row (+2, +3, +4...). |

#### Cognitive Level 2: Execute
| Field | Value |
|---|---|
| **Task type** | Near calculation |
| **Repetition Arc** | **Execution (x5):** Linear growing patterns: e.g., Step 1 has 3 dots, Step 2 has 5 dots, Step 3 has 7 dots. "Draw step 4. How many dots in step 5?"<br>**Endurance (x2):** "A plant grows 2 leaves every week. It started with 3. Make a table for weeks 1 to 5." |
| **Parent Prompt** | "Show the pattern. Ask your child to build or draw the next two steps, and then count the pieces." |
| **Success** | Accurately extends the pattern and identifies the constant rate of change (e.g., adding 2 each time). |

#### Cognitive Level 3: Discern
| Field | Value |
|---|---|
| **Task type** | Far calculation / Rule finding |
| **Parent Prompt** | "Show a pattern that starts with 2 and adds 3 each time (2, 5, 8, 11...). Ask: 'Without drawing the whole thing, how many would be in step 10? How do you know?'" |
| **Success** | Starts linking the step number to the total (e.g., "10 steps means I added 3 ten times, so 30, plus the 2 I started with" - intuitive y=mx+b). |

#### Cognitive Level 4: Own
| Field | Value |
|---|---|
| **Task type** | Pattern generation |
| **Parent Prompt** | "Say: 'Design a growing pattern out of blocks where the 5th step has exactly 17 blocks. Draw it or build it for me.'" |
| **Success** | Works backwards from the constraint to design a consistent linear pattern (e.g., starts with 1, adds 4 each step: 1, 5, 9, 13, 17 or starts with 5, adds 3: 5, 8, 11, 14, 17). |
| **Reasoning check** | "How do you know your pattern is the ONLY one that works? Could someone else design a different pattern that also reaches 17 at step 5?" |

## Formal Milestone Tasks — Strand 2 (All Capacities)

*Per Part VIe of the Spine: every milestone is a real-world situation with no mathematical vocabulary signalling the operation.*

---

### B2 Milestone: Unknown Quantities
> **Situation:** "Tendo went to the market with some money. He came back with 500 UGX change and he bought groundnuts for 1,500 UGX. How much did he go with?"
>
> **Success:** Child recognises this is a missing-start problem (?  - 1500 = 500). Finds 2000. Reasoning: explains how they worked backwards.

### B3 Milestone: Function Machines
> **Situation:** "Every week, Amara's sunflower grows taller. In week 1, it was 5 cm. In week 2, it was 8 cm. In week 3, it was 11 cm. Her grandmother asks: 'How tall will it be in week 6 without measuring?'"
>
> **Success:** Child identifies the constant growth rate (+3 per week), extends the table, and predicts 20 cm. Reasoning: explains the rule they discovered.

### B4 Milestone: Generalization
> **Situation:** "Your friend says 'If I add any two numbers and then subtract one of them, I always get the other one back.' Is that always true? How would you convince someone who doesn't believe it?"
>
> **Success:** Child tests multiple cases, then attempts a general argument ("because adding and then taking away undoes itself"). Does not need to be told this is about inverse operations.

### B5 Milestone: Growing Patterns
> **Situation:** "Nala is building a fence. Each section needs 4 posts. But the sections share posts — the last post of one section is the first post of the next. She has 25 posts. How many complete sections can she build?"
>
> **Success:** Child recognises the growing pattern (1st section = 4 posts, each additional section = 3 new posts). Calculates sections:  4 + 3(n-1) ≤ 25 → 8 sections. Does not need "algebra" vocabulary.

| B2 Discern | Discern | "Why can't Triangle be 12 if Triangle + Square = 10? What rule is being broken?" |

| B3 Execute | Execute | "How do you know your rule works for EVERY row, not just the ones you checked?" |
| B3 Discern | Discern | "What's the minimum number of examples you need to be SURE of the rule? Is 1 enough? Is 2?" |
| B3 Own | Own | "How do you know your machine has only ONE possible rule? Could there be a different rule that gives the same outputs?" |
| B4 Encounter | Encounter | "Did it work for every number you tried? Does that prove it works for ALL numbers? Why or why not?" |
| B4 Execute | Execute | "You tested 5 pairs. Is 5 enough to prove it's ALWAYS true? What would be enough?" |
| B4 Discern | Discern | "You found one counterexample. Is one enough to prove the claim is false? Why?" |
| B4 Own | Own | "How do you know your calendar rule works for every 2×2 square, not just the ones you tried?" |
| B5 Encounter | Encounter | "What do you notice about how many NEW blocks you add each time? Is it always the same?" |
| B5 Execute | Execute | "If I told you step 100, could you figure it out without drawing all 100 steps? How?" |
| B5 Discern | Discern | "How did you find step 10 without building every step? What shortcut did you use?" |
